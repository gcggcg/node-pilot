**需求：现有管理页面支持分页**

基于gitnexus知识图谱项目的现有代码结构和前端分页组件实现方式，为以下三个管理页面实现统一规范的分页功能：

1. **服务器管理页面**：实现列表分页，默认每页10条记录
2. **脚本管理页面**：实现列表分页，默认每页10条记录
3. **任务管理页面**：实现列表分页，默认每页10条记录

**分页功能要求**：

- 采用与gitnexus项目一致的ant-design分页组件和样式规范
- 分页组件需包含：首页、末页、上一页、下一页、跳页输入框、每页条数选择器（10/20/50/100）
- 支持总记录数显示和跳转至指定页
- 分页参数通过URL query参数同步（如page、pageSize）
- 后端接口需支持page、pageSize分页参数，返回total总数和data数组
- 页面初始化时自动加载第一页数据
- 分页切换时保留筛选和排序状态
- 加载状态需显示loading动画，分页数据为空时显示空状态提示

**开发参考**：直接复用gitnexus项目中已实现的分页组件代码和后端分页接口模式，确保三个页面的分页功能和交互体验保持一致。

## 维护记录

### Bug Fix 1: 分页组件刷新后消失 (2026-04-01)

**问题描述**：分页组件在页面刷新后有时显示有时不显示，具有不确定性。

**根本原因**：竞态条件 (Race Condition)

在 `loadData()` 函数中，`fetchTasks()` 是异步调用但未被正确等待：

```typescript
// 错误 ❌ - fetchTasks() 是 Promise，但未 await
function loadData() {
    store.fetchTasks(pagination.value.page, pagination.value.pageSize);
    pagination.value.total = store.pagination.total;  // 此时数据还没返回，total 是旧值 0
}
```

由于 `store.pagination.total` 在数据返回前就被读取，导致本地 `pagination.total` 被设为 0（或其他旧值），而 `v-if="pagination.total > 0"` 条件不满足，分页组件被隐藏。

**修复方案**：为 `loadData()` 添加 `async/await` 等待数据返回后再读取 total：

```typescript
// 正确 ✅
async function loadData() {
    await store.fetchTasks(pagination.value.page, pagination.value.pageSize);
    pagination.value.total = store.pagination.total;  // 数据返回后才读取
}
```

**涉及文件**：
- `frontend/src/views/ServerList.vue`
- `frontend/src/views/ScriptList.vue`
- `frontend/src/views/TaskList.vue`

### Bug Fix 2: 删除操作后总数不更新 (2026-04-01)

**问题描述**：在三个管理页面（服务器/脚本/任务）进行删除操作后，页面显示的总记录数没有变化，需要手动刷新页面才能看到更新后的数量。

**根本原因**：本地 pagination 状态未同步

删除操作调用 `store.deleteXxx()`，该方法内部会调用 `fetchXxx()` 刷新 store 中的数据并更新 `store.pagination.total`。但视图层的本地 `pagination.value.total` 从未被同步更新。

**修复方案**：在所有删除操作完成后，主动从 store 同步 `pagination.total`：

```typescript
// 修复后 ✅
async function deleteServer(id: number) {
    if (confirm('确定要删除此服务器吗？')) {
        await store.deleteServer(id);
        selectedIds.value = selectedIds.value.filter(i => i !== id);
        pagination.value.total = store.pagination.total;  // 同步总数
    }
}

async function deleteSelected() {
    if (confirm(`确定要删除已选的 ${selectedIds.value.length} 个服务器吗？')) {
        await store.deleteServers(selectedIds.value);
        selectedIds.value = [];
        pagination.value.total = store.pagination.total;  // 同步总数
    }
}
```

**涉及文件**：
- `frontend/src/views/ServerList.vue` - `deleteServer()`, `deleteSelected()`
- `frontend/src/views/ScriptList.vue` - `deleteScript()`, `deleteSelected()`
- `frontend/src/views/TaskList.vue` - `cancelTask()`, `deleteSelected()`

### Vue 3 组件规范提醒

**v-model 不能直接用于 props**：Vue 3 中 props 是只读的，不能直接使用 `v-model` 绑定 prop。

错误示例：
```vue
<!-- 错误 ❌ - v-model cannot be used on a prop -->
<select v-model="pageSize" ...>
```

正确做法是使用 computed 包装：
```vue
<!-- 正确 ✅ -->
<script setup>
const localPageSize = computed({
    get: () => props.pageSize,
    set: (val) => emit('update:pageSize', val)
});
</script>
<select v-model="localPageSize" ...>
```

参考组件：`frontend/src/components/Pagination.vue`
