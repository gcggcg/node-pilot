# Node-Pilot 批量服务器管理平台 - 登录页面

基于Go + Gin后端 + Vue 3 + TypeScript + Vite前端技术栈，实现一个具有现代科技感的服务器管理平台登录页面，具体要求如下：

## 登录页面设计

- 采用深色主题背景，模拟终端/控制台风格，添加动态粒子效果或数据流光晕动画
- 表单采用玻璃拟态(Glassmorphism)设计风格，边框带有发光效果，呈现科幻控制台质感
- 输入框采用霓虹灯风格，聚焦时有光晕扩散动画，模拟命令行输入体验
- 登录按钮采用渐变色，悬停时显示流光动效，带有"启动连接"或"接入系统"的视觉暗示
- 页面整体响应式设计，适配桌面和移动端，可融入服务器状态指示器、数据流动画等元素
- 可添加系统启动画面、加载动画等服务器管理平台风格的视觉元素

## 认证与权限控制

- 使用JWT Token进行身份认证，Token有效期设置为24小时
- Token存储于localStorage，请求时通过Authorization Header传递（格式：`Bearer {token}`）
- 登录成功后后端返回access_token和refresh_token
- 后续所有的操作都需要默认添加该认证方式，避免未授权直接访问
-

## 默认用户初始化

- 系统启动时（应用初始化时）自动创建root用户，用户名和密码均为`root`
- root用户角色标识为`ROLE_ADMIN`，其他用户角色为`ROLE_USER`

## 登录后功能

### 所有已登录用户可操作

- 修改个人信息：用户名、邮箱、联系电话、密码
- 查看当前登录状态和基本信息

### 仅root用户可操作

- 用户管理：添加新用户（需指定用户名、初始密码、角色）
- 用户删除：列表展示所有用户，支持单选/批量删除
- 用户查询：分页展示用户列表，支持按用户名模糊搜索

## 接口设计

后端API统一前缀为`/api/v1`，Go后端采用Gin框架实现RESTful风格接口：

- `POST /api/v1/auth/login` - 用户登录，返回JWT（access_token与refresh_token）
- `GET /api/v1/auth/me` - 获取当前用户信息（需认证）
- `PUT /api/v1/user/profile` - 修改个人信息（需认证）
- `GET /api/v1/admin/users` - 获取用户列表，分页参数`page`/`size`，搜索参数`keyword`（需管理员权限）
- `POST /api/v1/admin/users` - 添加用户，包含username/password/role字段（需管理员权限）
- `DELETE /api/v1/admin/users/{id}` - 删除用户，支持批量删除（id用逗号分隔）（需管理员权限）

## 安全要求

- 敏感接口需携带有效JWT Token，通过Gin中间件进行Token验证
- 删除操作用户确认对话框，防止误操作
- 用户密码需加密存储（bcrypt）
- 接口权限校验：非管理员用户访问管理接口返回403 Forbidden，需在中间件中校验用户角色
- 敏感操作日志记录，便于审计
