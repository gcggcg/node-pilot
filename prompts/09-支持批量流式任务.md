## 一、项目背景

基于现有的知识图谱依赖库列表./依赖库.md ,进行任务管理功能的扩展开发。项目涉及任务管理模块的脚本执行能力增强，需要支持批量脚本的顺序执行与目录切换逻辑。

## 二、需求描述

### 2.1 批量脚本添加功能

在任务管理页面中，新增/编辑任务时增加批量脚本添加功能模块，具体要求如下：

- **输入方式**：支持通过界面多选按键选择多个脚本添加。
- **顺序保证**：脚本按用户在界面中添加/填写的先后顺序依次执行
- **输入示例**：`/path/to/script1.sh, /path/to/script2.sh, /path/to/script3.sh`

### 2.2 流式执行与错误处理

脚本执行引擎需实现以下逻辑：

- **顺序执行**：严格按照添加顺序逐个执行脚本，前一个脚本执行完成后（并且成功）再执行下一个
- **错误终止**：当任意脚本执行失败时，立即终止后续所有脚本的执行流程
- **异常返回**：错误发生时返回以下信息：
    - 标识当前是第几个脚本执行失败（如：第 2/5 个脚本执行失败）
    - 失败脚本的完整路径或脚本名称
    - 具体错误描述/异常堆栈

### 2.3 目录切换执行机制

修改现有脚本执行方式，具体要求如下：

**原执行方式（需废弃）**：

```
execCmd := fmt.Sprintf("/bin/bash %s", targetFile)
cmdOut, err := session.CombinedOutput(execCmd)
```

**新执行方式（需实现）**：

- 获取目标脚本文件的所在目录路径（`targetFile` 的父目录）
- 先执行 `cd` 命令切换到脚本所在目录
- 再在目标目录下执行该脚本
- 执行命令示例：

```
cd /path/to/script/parent/directory && /bin/bash script_name.sh
```

- **实现原因**：确保脚本执行时能够正确访问同目录下的依赖文件和相对路径资源

## 三、依赖知识图谱管理

- 知识图谱依赖库列表文件：`./依赖库.md`
- 需查阅该文件确认现有的任务管理模块相关依赖和执行框架
- 重点关注脚本执行相关的会话管理（session）模块的接口定义

## 四、数据源

- **任务配置数据表**：存储任务配置信息，包含批量脚本路径字段（建议使用 `TEXT` 类型存储逗号分隔的脚本路径列表）
- **执行日志表**：记录每个脚本的执行状态、开始时间、结束时间、输出结果、错误信息

## 五、业务核心逻辑

### 5.1 流式执行引擎逻辑

```
FOR i := 0 TO len(脚本队列) - 1:
    当前脚本 := 脚本队列[i]
    
    获取脚本所在目录 := filepath.Dir(当前脚本)
    构建执行命令 := "cd " + 脚本所在目录 + " && /bin/bash " + 当前脚本
    
    执行命令并捕获输出和错误
    IF 执行成功:
        记录执行日志，继续下一个脚本
    IF 执行失败:
        记录错误日志
        返回错误响应，包含：
            - 当前执行位置（i+1）和总数量（len(脚本队列)）
            - 失败脚本路径或脚本名称
            - 错误信息
            
        终止循环，不继续执行后续脚本
```

### 5.2 异常处理规则

| 异常类型        | 处理方式 | 返回信息                |
|-------------|------|---------------------|
| 脚本路径不存在     | 终止执行 | 提示脚本文件不存在           |
| 脚本无执行权限     | 终止执行 | 提示权限不足              |
| 脚本执行超时      | 终止执行 | 提示执行超时              |
| 脚本执行返回非零状态码 | 终止执行 | 输出脚本stdout/stderr内容 |

## 六、开发约束

### 6.1 技术约束

- **会话管理**：使用现有的 `session` 模块进行命令执行
- **命令构造**：必须采用 `cd <目录> && /bin/bash <脚本>` 格式，禁止使用原有的直接执行方式
- **路径处理**：使用 `path/filepath` 包处理路径，确保跨平台兼容性

### 6.2 代码规范

- 错误信息需包含脚本索引位置（从 1 开始计数）
- 所有日志输出需包含时间戳和脚本路径
- 代码中需添加注释说明目录切换的执行逻辑

### 6.3 兼容性要求

- 保持现有单脚本执行功能的向后兼容
- 新增的批量功能作为可选项，不影响原有任务创建流程
