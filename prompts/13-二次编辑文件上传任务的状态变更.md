# 需求文档

## 1. 需求概括

修复文件上传任务的状态显示问题：当通过编辑页面修改并保存已执行过的文件上传任务时，将任务状态统一重置为"待执行"(pending)，避免用户对任务实际执行状态产生混淆。

## 2. 需求详情

### 功能点列表

1. **[编辑保存时状态重置]**
    - 输入：用户编辑已有文件上传任务（该任务当前状态为 success 或 failed）并点击保存
    - 处理逻辑：在 `UpdateFileUpload` 方法中，更新任务主记录时强制将 status 字段重置为 "pending"
    - 输出：任务状态更新为 "pending"，关联的 file_upload_servers 记录状态也需同步重置为 "pending"

2. **[关联服务器状态同步重置]**
    - 输入：编辑保存时重置主任务状态
    - 处理逻辑：更新主记录状态后，同时将对应的 file_upload_servers 关联记录的状态也重置为 "pending"，清空之前的执行结果（error_message 等）
    - 输出：关联的服务器上传记录状态均为 "pending"，准备重新执行

### 边界条件与异常处理

- **条件1**：任务状态为 "running"（正在执行中）时不允许编辑
    - 处理方式：在 UpdateFileUpload handler 中增加状态校验，若为 running 返回错误提示

- **条件2**：编辑时未修改任何内容直接保存
    - 处理方式：仍按正常流程重置状态为 pending

- **条件3**：编辑后只更新了部分字段（如只改了 name）
    - 处理方式：只要触发保存操作，就必须重置状态为 pending

## 5. 需求核心处理逻辑

### 主流程

```
用户编辑文件上传任务 → 点击保存 → 前端调用 UPDATE API
                                            │
                                            ▼
                              ┌─────────────────────────────┐
                              │   Backend: UpdateFileUpload │
                              │   - 校验任务状态 != running  │
                              │   - 更新 name/path/server_ids│
                              │   - 强制 status = "pending"  │
                              │   - 删除旧关联，重建关联     │
                              │   - 关联状态也重置为 pending │
                              └─────────────────────────────┘
                                            │
                                            ▼
                              ┌─────────────────────────────┐
                              │      返回更新成功            │
                              └─────────────────────────────┘
```

### 关键步骤说明

1. **[步骤1：状态校验]** - 检查任务当前状态是否为 running，若是则拒绝编辑
2. **[步骤2：更新主记录]** - 更新 name、local_path、remote_path，同时强制设置 status = "pending"
3. **[步骤3：重建服务器关联]** - 删除旧的 file_upload_servers 记录，新建关联并设置 status = "pending"
4. **[步骤4：返回响应]** - 返回更新成功状态

### 性能考虑

- 删除和创建关联记录在同一事务中处理，确保数据一致性
- 无显著性能瓶颈，涉及数据量较小

## 6. 测试要求

### 验收标准

1. 编辑一个状态为 "success" 的文件上传任务，保存后状态变为 "pending"
2. 编辑一个状态为 "failed" 的文件上传任务，保存后状态变为 "pending"
3. 编辑一个状态为 "pending" 的文件上传任务，保存后状态保持 "pending"
4. 编辑一个状态为 "running" 的文件上传任务，返回错误提示"任务正在执行中，无法编辑"
5. 编辑保存后，关联的服务器上传记录状态也重置为 "pending"，之前的错误信息被清空
