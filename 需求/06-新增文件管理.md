**需求：文件上传管理功能**

基于GitNexus知识图谱中当前项目的整体设计方式，实现文件上传管理页面，参考脚本管理页面的交互设计和代码实现。

## 一、页面结构与布局

- **整体布局**：采用与脚本管理页面一致的上下结构（查询筛选区 + 数据列表区 + 操作按钮区）
- **列表字段**：序号、上传文件名、本地路径（./data/files/下的相对路径）、目标服务器列表、远程路径、上传状态（成功&#x2F;失败&#x2F;待执行）、创建时间、操作列
- **操作列按钮**：编辑、执行、查看结果、删除
- **顶部按钮**：新增上传配置

## 二、新增上传配置功能

1. **本地文件选择**：
    - 触发方式：点击按钮弹出系统文件选择器（input[type&#x3D;file] multiple）
    - 数量限制：单次最多选择20个文件
    - 文件格式：根据后缀自动识别，支持.tar、.sh、.zip、.conf、.txt、.json、.yml、.xml、.log、.sql共10种格式
    - 其他格式文件：提示“不支持的文件格式”，阻止选择
    - **文件存储**：用户选择文件后，自动将文件保存至项目目录下的 ./data/files/ 目录中，按原文件名存储；若文件已存在则覆盖

2. **目标服务器选择**：
    - 组件：多选下拉框
    - 数据源：调用系统已有的服务器列表接口
    - 选择上限：最多选择10台服务器
    - 选中展示：每台服务器显示 IP:PORT 格式

3. **远程路径配置**：
    - 输入方式：文本输入框
    - 格式要求：必须以&#x2F;开头（如 &#x2F;data&#x2F;upload&#x2F;）
    - 校验：路径为空或格式错误时禁用保存按钮
    - 作用：所有选中服务器使用相同的远程路径

4. **保存行为**：
    - 点击保存后，将文件路径（./data/files/下的相对路径）、服务器列表、远程路径写入数据库
    - 上传状态默认为"待执行"
    - 不触发实际上传操作
    - 成功后关闭弹窗并刷新列表

## 三、编辑功能

- 点击编辑后弹出编辑弹窗
- 可修改项：目标服务器选择、远程路径
- 不可修改项：本地文件列表（需删除后重新新增）
- 保存后更新数据库记录

## 四、执行上传功能

1. **触发方式**：点击操作列的“执行”按钮
2. **执行范围**：读取数据库中记录的本地路径，从 ./data/files/ 目录获取实际文件，上传至当前配置指定的目标服务器
3. **上传方式**：遍历目标服务器列表并行执行（参考任务管理的执行方式）
4. **上传结果**：
    - 每台服务器生成一条独立的执行结果记录
    - 状态标记为：上传成功 &#x2F; 上传失败
    - 失败记录需记录具体错误信息（连接失败、权限不足、路径不存在等）

## 五、结果查看功能

1. **触发方式**：点击操作列的“查看结果”按钮
2. **弹窗内容**：
    - 标题显示：配置名称 + 执行时间
    - 结果列表：
        - 服务器信息（IP:PORT）
        - 上传文件名
        - 状态（成功图标&#x2F;失败图标 + 文字）
        - 失败原因（仅失败项显示）
    - 每条记录显示文件级别详情：文件名 → 目标路径 → 状态

## 六、批量删除功能

1. **勾选方式**：列表左侧增加复选框（参考脚本管理页面的批量勾选交互）
2. **全选功能**：表头复选框实现全选&#x2F;取消全选
3. **删除范围**：可删除成功、失败、待执行状态的记录
4. **确认机制**：删除前弹出确认框，显示即将删除的记录数量
5. **执行后**：刷新列表，更新分页信息
6. **文件清理**：删除记录时同步删除 ./data/files/ 目录下对应的文件

## 七、分页功能

1. **分页参数**：page（页码）、pageSize（每页条数）
2. **默认配置**：pageSize&#x3D;10
3. **页码导航**：首页、上一页、下一页、末页 + 直接跳转输入框（参考脚本管理页面的分页组件）
4. **总记录数**：列表底部显示“共 X 条记录”

## 八、查询筛选功能

- **筛选条件**：
    - 上传状态：全部&#x2F;待执行&#x2F;成功&#x2F;失败
    - 文件名：模糊搜索
    - 日期范围：创建时间起止（可选）
- **查询按钮**：点击后刷新列表数据
- **重置按钮**：清空筛选条件，恢复默认列表

## 九、错误处理

1. **文件上传失败**：记录失败原因，显示在结果详情中
2. **网络异常**：提示“网络连接失败，请重试”
3. **文件过大**：前端限制单文件不超过500MB，超出提示“文件超过大小限制”
4. **权限不足**：失败结果中明确提示权限错误信息
5. **文件不存在**：执行时检查 ./data/files/ 目录，若文件不存在则提示"本地文件不存在，请重新上传"

## 十、接口要求

- **列表接口**：GET &#x2F;api&#x2F;file-upload&#x2F;list，参数包括page、pageSize、status、fileName、startTime、endTime
- **新增接口**：POST &#x2F;api&#x2F;file-upload，参数包括fileList（文件路径列表，相对于./data/files/）、serverIds、remotePath
- **编辑接口**：PUT &#x2F;api&#x2F;file-upload&#x2F;{id}
- **执行接口**：POST &#x2F;api&#x2F;file-upload&#x2F;{id}&#x2F;execute
- **结果查询接口**：GET &#x2F;api&#x2F;file-upload&#x2F;{id}&#x2F;results
- **删除接口**：DELETE &#x2F;api&#x2F;file-upload，参数为ids数组
- **服务器列表接口**：GET &#x2F;api&#x2F;servers（复用于现有接口）

## 十一、数据存储说明

- **文件存储位置**：所有上传的本地文件统一存储在项目根目录下的 ./data/files/ 目录
- **数据库记录**：仅存储文件路径（相对于 ./data/files/ 的路径），不存储文件二进制内容
- **路径格式**：数据库中记录的本地路径格式为相对路径，如 "scripts/deploy.sh" 表示实际文件位于
  ./data/files/scripts/deploy.sh
